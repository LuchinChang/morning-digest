📚 Deep Dive (20 min)

AgentSpec: Customizable Runtime Enforcement for Safe and Reliable LLM Agents

DEEP DIVE
ICSE 2026 (April, Rio de Janeiro) ~20 min Formal methods × agents

A lightweight DSL for specifying and enforcing runtime safety constraints on LLM agents. Each AgentSpec rule is a three-tuple: triggering event, predicate functions, and enforcement actions. The framework supports both preventive (block before execution) and corrective (repair after detection) enforcement modes. Results: 90%+ unsafe execution prevention in code agents, 100% compliance in autonomous driving scenarios, with overhead in milliseconds. The automated rule generation feature — using LLMs to write safety specs for LLMs — is the most thought-provoking part.

This is exactly your research neighborhood: formal specification meets learned agents. The DSL design choices (expressiveness vs. verifiability tradeoffs) and the preventive-vs-corrective modes map directly onto the shielding literature. Read Sections 1–3 carefully, skim Section 5 (evaluation).

Karpathy: "I've Never Felt This Behind as a Programmer"

TRENDING
Andrej Karpathy / X ~4 min Viral thread — Hacker News front page

Karpathy admits the profession is being "dramatically refactored" — programmers now contribute increasingly sparse bits between AI-generated code. His key insight: you could be 10X more powerful if you properly string together what's become available, and failing to do so is "a skill issue." He describes a new programmable layer of abstraction: managing agents, subagents, prompts, memory, permissions, and tools. Not doom — an invitation to level up.

Research philosophy from the trenches. The "sparse and between" framing applies to PhD research too: your value increasingly lies in knowing what to ask the tools to do, not in grinding through every step yourself.

NASA Perseverance Completes First AI-Planned Drive on Mars

TRENDING
NASA / JPL × Anthropic ~4 min Deployed AI agents — literally on another planet

Perseverance drove 689 feet on Dec 8 and 807 feet on Dec 10, 2025 — the first Mars drives ever planned by AI. JPL used Claude's vision-language models to analyze terrain data and stitch together waypoints, replacing a planning task humans had done manually for 28 years. Safety verification: commands were run through JPL's digital twin, checking 500,000+ telemetry variables before transmission. The formal verification angle writes itself — this is runtime verification of an autonomous agent in the highest-stakes environment imaginable.

Real-world deployment of exactly the pattern AgentSpec formalizes: agent proposes actions, verification layer checks them, only safe commands execute. Except the stakes are a $2.7B rover on Mars.

Shields for Safe Reinforcement Learning

READ
Könighofer, Bloem, Jansen, Junges & Pranger / CACM 68(11) ~5 min (intro + framework overview) Survey — formal methods × RL

A comprehensive review of shielding for safe RL: external modules that block unsafe actions at runtime based on a formal model of the world. Covers discrete and continuous settings, model-free and model-based RL, single-agent and multi-agent scenarios. The key insight: shielding decouples safety logic from reward-driven policy optimization, combining symbolic AI's provable guarantees with RL's scalability. Now published in Communications of the ACM — a signal that the mainstream CS community considers this approach mature enough for broad attention.

Core reading for your research area. This survey is the canonical reference for shielded RL. Skim the intro and Section 2 (framework overview) today; bookmark for a deeper multi-day pass later if needed.
